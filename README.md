# 新文化式原型產生器

依日本文化服裝學院「新文化式(2000年版)成人女子原型」製圖法,輸入 **胸圍 B、腰圍 W、背長**(袖原型另需袖長),自動產生上半身原型與袖原型,可下載 SVG 與 1:1 實寸向量 PDF。

- 已部署:https://8kvzfyz59n-sudo.github.io/hell/ (repo: 8kvzfyz59n-sudo/hell, GitHub Pages, main branch root)
- 目前版本/功能完成度/待辦 → 見 [`STATUS.md`](STATUS.md)

## 使用方式

直接用瀏覽器開啟 `index.html` 即可,完全離線可用(PDF 由內建向量產生器輸出,不依賴外部函式庫,除首頁 3D 需連網)。
要公開部署,把整個資料夾上傳到任何靜態空間(GitHub Pages、Cloudflare Pages、Netlify)即可。

## 功能

上半身原型、袖原型、直筒裙(含隱形拉鍊/開衩/腰頭記號)、圓裙(1/4~全圓)、直筒褲、百褶裙用布量計算機、男裝上衣原型(身片+袖)、男裝西裝褲 + 各分頁教學區塊。PDF 一次輸出多頁 1:1 實寸。

|---|---|
| 衣寬 | C/2 + 6.7 |
| A~BL(袖窿深) | C/6 + 8.5 |
| 背幅 | C/6 + 5.8 |
| 胸幅 | C/6 + 2.9 |
| 後中心長 | 背長 + 0.5 |
| 前領寬 | C/16 + 1.9 |
| 前領深 | 前領寬 + 0.5 |
| 後領寬 | 前領寬 + 0.3 |
| 前/後肩斜 | 22° / 21° |
| 後肩褶 | C/32(指向 I 點=背幅中段) |
| 總腰省 | (C/2+6.7) − (W/2+4),分配 a16/b16/c36/d24/e8(%) |

## 百褶裙用布量計算(陽折/陰折,三摺型)

陽折=表面看得到的褶寬;陰折=摺疊進去、藏在下面的深度。摺型公式(v13,以布料路徑模擬推導):

| 摺型 | 每褶用布 | 腰上佔寬 | 每褶摺線(自褶起點) |
|---|---|---|---|
| 刀褶 | 陽+2陰 | 陽 | 山@陽、谷@陽+陰 |
| 箱褶(工字) | 陽+4陰 | 陽 | 山@陽/2、谷@陽/2+陰、谷@陽/2+3陰、山@陽/2+4陰 |
| 手風琴褶 | 陽+陰 | 陽−陰(陽≤陰時無法平面圍腰,顯示警告) | 山@陽、谷@陽+陰 |

褶數=無條件進位(淨腰圍+鬆份)÷腰上佔寬;總裙寬=褶數×每褶用布+後開口/接縫份;裙長方向所需寬度=裙長+腰頭寬+下擺縫份+腰圍縫份,需≤布幅才能橫裁不拼接。

**A4 試摺紙型**(pleatSheetSVG):21×29.7cm SVG 直出 PDF 即 1:1 A4;單元寬>19 自動轉橫式、>27.7 回傳 null(頁面顯示提示)。山摺=紅點劃線、谷摺=藍虛線,含 10cm 列印校正尺標/摺向箭頭/摺完寬度對照條/每褶編號;箱褶採工字褶排列(一般箱褶=翻面摺)。
註:v10~v12 的「摺份剖面圖」(pleatSVG)因摺型畫法錯誤,2026-07-08 經使用者確認整個移除。

## 直筒裙公式(簡化文化式)

半身寬 H/2+2、脇線=HL中點後移1cm、後腰目標 W/4、前腰目標 W/4+1;縮減量一半收脇邊、一半做兩個腰省;拉鍊開口=腰長+1、後開衩=裙長/3、腰頭 W+3×3cm。

## 圓裙公式(圓周率法)

n/4 圓裙腰半徑 r=2W/(nπ),下襬半徑=r+裙長;版片畫半件(圓心角 n×45°)。

## 架構要點(改程式前必讀)

1. **座標系**:cm 為單位,y 向下。SVG 直接用 cm 作 viewBox 單位,width/height 屬性帶 cm → SVG 本身就是實寸。
2. **PDF 是自製產生器**(app.js 內 svgToPdfPage/buildPdf),不用任何函式庫。限制:
   - path 只支援 M + C(貝茲)指令——畫直線用 lineC()、圓弧用 arcC() 轉成退化/近似貝茲,**絕不能在 SVG path 用 L/A/Q 指令**
   - **文字只支援 ASCII(Helvetica),圖面標籤禁用中文**——凡是會被 dlPDF()/dlOnePdf() 轉檔的 SVG,文字一律只能用 ASCII,中文說明改放在 tool.html 頁面文字
   - 解析同時支援 `<line/>` 與 `<line></line>` 兩種序列化(歷史 bug:瀏覽器 outerHTML 會改寫自閉合標籤,曾導致 PDF 只有文字沒線條)
3. **下載一律用原始 SVG 字串**(cur.xxxSvg),不要從 DOM outerHTML 取。
4. **分頁錨點**:#top/#tight/#circle/#pants,activateTab() 處理;輸入欄用 data-tabs 屬性綁定顯示。
5. **快取**:script src 帶 ?v=N,每次改版遞增並同步改 ver 標記。
6. 曲線平滑用 Catmull-Rom(crPathD),曲線長用數值積分(crLen)。
7. **網頁上不顯示資料來源**(使用者要求,2026-07-08 已移除 tool.html 來源區塊):新增分頁時不要再放來源連結;查證依據記在本 README「來源」段即可。
8. pleat.js 的 A4 紙型全用 `<line>`/`<text>` 元素(無 path),樣式一律放元素屬性——PDF 轉換器不解析 `<g>` 繼承樣式,也是舊剖面圖 PDF 壞掉(NaN)的原因之一。

## 已知限制

- 領口、袖窿、袖山弧線以教材導引點 + 平滑曲線近似,與手繪 D 彎尺結果會有毫米級差異。
- B ≥ 90 時胸省閉合後前袖窿會出角,教材要求手動修順(頁面會提示)。
- 新文化式袖山縮縫份(いせ)本來就偏多(約 6~8%),縫製時需以兩道疏縫抽縮處理。
- 直筒裙省長(後13/12前9/10)與褲省長是教科書定值,特殊體型需手調。
- 男子原型公開資料較少:腰省位置/前身上端為本站近似;男褲公式為通行慣例彙整,無單一權威教材,務必胚布試穿補正。
- 原型僅為打版基礎,實際商品開發仍需胚衣試穿補正。
