在既有新文化式打版網站新增男裝功能:輸入尺寸即可產生男裝上衣原型(身片+袖)與男裝西裝褲版型,輸出 SVG 與 1:1 實寸 PDF。首頁與打版圖視覺風格完全沿用現有樣式。

背景:男裝功能曾於 v9 時被使用者要求停止並刪除;本次為使用者**主動重新提出**,符合 STATUS.md 記載的恢復條件。

## 2. 男裝上衣原型模組(men.js)

**輸入**(預設=教材例):胸圍 C=92、腰圍 W=74、背長=45、袖長=60(cm)。

**公式主幹**(來源:愛打版 iPatterning 新文化式男子原型 https://ipatterning.com/menprototype/):

| 部位 | 公式 |
|---|---|
| 後中心長 | 背長 + 0.5 |
| 衣寬 | C/2 + 6.7 |
| 袖窿深(A~BL) | C/6 + 8.5 |
| 背寬 | C/6 + 5.8 |
| 胸寬 | C/6 + 2.9 |
| 前領寬 | C/16 + 1.9 |
| 前領深 | 前領寬 + 0.5 |
| 後領寬 | 前領寬 + 0.3 |
| 前/後肩斜 | 22° / 21° |
| 肩省 | C/32 |
| 總腰省 | (C/2+6.7) − (W/2+4),分配 a~e:16/16/36/24/8% |
| 袖山高 | (SP~BL) × 5/6 |
| 前袖斜線 | 前AH |
| 後袖斜線 | 後AH + 1 |
| 肘線 | 袖長/2 + 2.5 |

實作前用第二個獨立來源交叉核對;對不上處在 README 註記取捨依據。

**輸出**:身片 SVG + 袖 SVG,畫風仿 `bodiceSVG`/`sleeveSVG`(黑線輪廓、灰輔助線、ASCII 標籤 BL/WL/SP/FAH/BAH、關鍵點圓點),進 PDF 多頁。

**曲線**:領口/袖窿/袖山用 `crPathD`(Catmull-Rom),AH 長用 `crLen` 數值積分。

**錯誤處理**:輸入範圍檢查(C 70~120 之外顯示提醒不繪圖);超出教材適用範圍時頁面中文提示(沿用女裝 B≥90 提示慣例)。

## 3. 男褲模組(menpants.js)

**輸入**:腰圍 W、臀圍 H、褲長、股上(預設「自動 = 褲長/10+18」,可手動)、褲口寬(選填,預設依 H 推算)。

**公式方向**(彙整中文西裝褲打版通行公式,實作前以 2~3 來源交叉驗證,來源記入 README):

- 前片臀圍 H/4−1、後片 H/4+1
- 小裆宽 ≈ H/20−1、後裆宽 ≈ H/10
- 落裆 1cm、后翘 2~2.5cm
- 燙跡線置於裆宽中點
- 前腰單褶 + 後腰省
- 中裆位置 = (褲長−股上) × 0.4,由此收窄至褲口

**輸出**:前片+後片 SVG(含燙跡線、腰省/褶記號、ASCII 標籤),進 PDF。

**驗收基準**:預設尺寸能畫出結構自洽的直筒西裝褲(前後裆弧順接、脇線/股下線等長邏輯成立)。

**待定值**(於實作計畫的「公式研究」步驟隨來源交叉驗證一併定案,不留到寫程式時才決定):褲口寬預設推算式、后翘取 2 或 2.5、股上自動式的常數(17~19 取一)、男褲預設輸入尺寸組。
