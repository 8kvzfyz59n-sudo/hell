# 百褶裙 A4 試摺紙型設計書(2026-07-08)

擴充第 5 分頁「百褶裙用布量」:新增摺型選擇(刀褶/箱褶/手風琴褶),依陽摺/陰摺/裙長推算整塊裙布所需寬度,並產生可 1:1 列印的 A4 試摺紙型,山摺/谷摺(凸/凹)清楚標示,讓使用者先摺紙確認褶型。

## 需求確認結果

| 項目 | 決定 |
|---|---|
| 摺型 | 刀褶、箱褶(工字褶)、手風琴褶三種都要 |
| 整合 | 擴充現有第 5 分頁(不另開分頁);陰摺/陽摺/裙長沿用既有輸入欄 |
| 紙型輔助 | 列印校正尺標、摺向箭頭、摺完寬度對照條、每褶編號(四項全要) |
| 輸出 | 沿用站內 SVG→自製 PDF 管線;紙型 SVG=21×29.7cm,PDF 頁即標準 A4 1:1(不用 window.print,避免印表機縮放不可控) |

## 1. 摺疊幾何與計算(pleat.js 擴充)

以布料路徑模擬推導,三摺型的常數:

| 摺型 type | 每褶用布 | 腰上佔寬 | 每褶摺線位置(自褶起點) |
|---|---|---|---|
| `knife` 刀褶 | 陽+2陰 | 陽 | 山摺@陽;谷摺@陽+陰 |
| `box` 箱褶(工字) | 陽+4陰 | 陽 | 山@陽/2、谷@陽/2+陰、(陽/2+2陰處無摺線)、谷@陽/2+3陰、山@陽/2+4陰(對稱一組,山摺線即表面兩摺邊相接處) |
| `accordion` 手風琴褶 | 陽+陰 | 陽−陰 | 山摺@陽;谷摺@陽+陰 |

- `draftPleatSkirt(...)` 加 `type` 參數('knife'|'box'|'accordion'),回傳增加 `type`、`waistPerPleat`(腰上佔寬):
  - 褶數 = 無條件進位(effWaist / waistPerPleat)
  - flatWidth = 褶數 × 每褶用布(其餘欄位邏輯不變)
- 驗證:accordion 且 陽 ≤ 陰 → `waistPerPleat ≤ 0`,draw() 顯示中文警告「手風琴褶陽摺需大於陰摺才能平面圍腰;陽=陰為純手風琴褶,需鬆緊帶或裙襬展開」,用布量欄位顯示警告、**試摺紙型照樣可出**(摺紙本身不受影響)。
- 新函式 `pleatSheetSVG(p)` → A4 試摺紙型 SVG 字串。
- **舊剖面圖整個移除**(使用者確認原摺型畫法是錯的):刪除 `pleatSVG` 函式與其 panel/下載按鈕,第 5 分頁圖形區只剩 A4 試摺紙型;`cur.plSvg` 由 `cur.plSheetSvg` 取代,「下載全部 PDF」中原剖面頁改為試摺紙型頁。(原剖面圖 PDF 輸出本來就含 NaN bug,隨刪除一併消失。)

## 2. A4 試摺紙型版面

- 尺寸:21×29.7cm 直式(viewBox 即 cm);單褶單元寬 > 19cm 時自動轉橫式(29.7×21,可用寬 27.7);仍放不下 → 頁面訊息「褶太寬無法放入 A4,請縮小數值」,不出圖。
- 邊界:四邊 1cm;摺線區高約 20cm(直式)。
- 內容(圖面文字全 ASCII;中文一律在 tool.html 頁面):
  1. 標題+參數:`PLEAT TEST SHEET - KNIFE`、`YANG 3 / YIN 2 / LEN 45`。
  2. 摺線:直向貫穿摺線區;**山摺=紅色點劃線**(#c0392b,dasharray 約 `0.6 0.25 0.08 0.25`)、**谷摺=藍色虛線**(#2467a8,dasharray 約 `0.5 0.3`)。能放幾個完整褶單元就畫幾個(至少 1)。
  3. 每褶編號:單元頂部 `1` `2` `3`…。
  4. 摺向箭頭:每條摺線區段上方小箭頭指示摺入方向(刀褶/手風琴:整段向左;箱褶:兩側向中心)。箭頭用 line+三角形線段(全部 lineC 相容)。
  5. 摺完寬度對照條:頂部一條標註線段,長=畫出褶數×腰上佔寬(+末端殘段),文字 `FOLDED WIDTH X cm`——摺好的紙應剛好蓋住這條。
  6. 列印校正尺標:底部 10cm 水平尺,逐 cm 刻度,文字 `PRINT CHECK: THIS BAR = 10 cm`。
  7. 圖例:兩種線樣本+`MOUNTAIN = FOLD OUT` / `VALLEY = FOLD IN`。
- SVG 限制照舊:path 只用 M+C(lineC),文字 ASCII,樣式放元素屬性(不靠 `<g>` 繼承)。

## 4. 測試

- preview_eval 數值:三摺型的褶數/flatWidth/waistPerPleat 對照手算;accordion 陽≤陰 警告觸發。
- 紙型 SVG:viewBox=`0 0 21 29.7`(或橫式)、ASCII、無 L/A/Q、svgToPdfPage 無 NaN、PDF 頭 `%PDF-1.4`;MediaBox ≈ 595×842pt(A4)。
- 既有 pleatSVG 修復後:svgToPdfPage(pleatSVG(...)).stream 無 NaN。
